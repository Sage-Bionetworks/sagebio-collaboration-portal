SBCP Documentation

Learn about the different types of information and actions in the portal.

Why a collaboration portal?

To facilitate collaboration, this portal aims to...

  • Put data and tools in the hands of life scientists
  • Connect quantitative scientists to biological and clinical domain expertise
  • Expand the audience for high-value data resources and tools
  • Help identify existing or alternative solutions to a common problem
  • Summarize and distill activity across data and tools, users and teams
  • Enable new connections and ideas around a common question
  • Accelerate discovery while capturing and clarifying attribution

Insights

Report An insight that would be characterized by 1) a potentially long description or "body" and 2) the option to include multiple Resource attachments (e.g., Files).
Memo Simpler version of a Report with a note referencing one or more Entities, but allowing only a single (or no) attachments.

Resources

Article Simple file with no specialized properties or behaviors. Usually an article that can be referenced/saved that is relevant to the project.
Dashboard Single-page, interactive web UI for exploring, visualizing, and summarizing pre-specified information.
State Intermediate or final state of a Tool session (i.e., a "bookmark"), with sufficient information to recreate/resume that state.
WebApp Like a web app/tool, but "static" (cannot load a new Dataset or Resource). Can be thought of as a multi-page/module dashboard.

Projects

Project Projects are spaces that can be public or private (shared with only your specific colalborators). Projects contain all the entities (Insights, Resources, Discussions) and they adopt the same sharing settings of their parent project.

Datasets

Datasets The portal allows you to query multiple data catalogs. The portal is connected to several different catalogs so that the user can query for their data of interest in one place then go to the data source.

Tools

Tool Analytical tools to explore, visualize, and manipulate different types of data.

Discussion

Discussion Discussions can happen in multiple ways. They can be general discussions that happen at the whole portal level, at the project level, and at the entity level.
